# E1: End-to-End Passthrough Integration Test

**Date:** 2026-07-21
**Branch:** `feat/extstore-e1-e2e-test`
**Status:** Complete

## Summary

Added an end-to-end integration test that exercises the full external store
passthrough flow through the gRPC API, using a mocked GitHub GraphQL server.
Also fixed the passthrough store's remote data to include `platform` and
`remote_url` fields so tasks are correctly identified as GitHub-sourced.

## Changes

### New files

- **`internal/server/passthrough_e2e_test.go`** — Three integration tests:
  - `TestPassthroughE2E` — Full flow: creates a GitHub collection + linked
    account in the primary store, wires up a MultiStore with a
    PlatformResolver, and verifies:
    - `ListTasks` triggers lazy resolution and returns 2 mock GitHub issues
    - Tasks have `platform=GITHUB`, populated `remote_id` and `remote_url`
    - Tasks have correct labels and collection association
    - `GetCollection` returns the GitHub platform
    - `ListLinkedAccounts` retrieves the linked account
    - `WatchTasks` returns `codes.Unimplemented` for the GitHub collection
    - Subsequent `ListTasks` calls use the cached platform store
  - `TestPassthroughE2E_NativeCollectionUnaffected` — Verifies farmtable
    collections still work normally with a PlatformResolver configured
  - `TestPassthroughE2E_LazyResolutionWithoutLinkedAccount` — Verifies
    fallback to primary store when no linked account exists

- **`internal/platform/github/testing.go`** — Exported `SetTestGraphQLClient`
  helper that allows test code to inject a mock `githubv4.Client` into a
  `GitHubPassThroughStore`.

### Modified files

- **`internal/platform/github/graphql_queries.go`** — Added `"platform":
  "github"` and `"remote_url"` fields to `issueBuildRemoteData`. This ensures
  `taskToProto` maps the task to `PLATFORM_GITHUB` instead of the default
  `PLATFORM_FARMTABLE`, and populates the `RemoteUrl` proto field.

- **`internal/testutil/testserver.go`** — Added `NewTestServerWithMultiStore`
  helper that creates a gRPC test server backed by a caller-provided
  `MultiStore` with streaming enabled.

## Test approach

The tests use **Option B** (mock GitHub API) for CI friendliness:
- An `httptest.Server` serves canned GitHub GraphQL responses
- `githubv4.NewEnterpriseClient` points at the mock server
- The mock is injected via `SetTestGraphQLClient` into a real
  `GitHubPassThroughStore`
- No real GitHub tokens or network calls are required

## Key assertions verified

| Assertion | Test |
|---|---|
| Collection created with platform=github | `GetCollection_returns_github_platform` |
| LinkedAccount created and associated | `LinkedAccount_retrievable` |
